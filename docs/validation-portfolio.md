# Validation Portfolio

Release: `0.5.0`

TESSARYN uses separate evidence layers because a visually strong synthetic
source and a noisy physical sensor source answer different questions. The
dataset source class is identity-bearing. A synthetic profile cannot be
relabelled as real sensor evidence without invalidating its selection manifest,
source Aggregate Cell, PHA binding, and Rootprint lineage.

## Exact-Ground-Truth Lab

The opt-in Validation Lab uses TartanAir V2:

- environment: `ArchVizTinyHouseDay`;
- sequence: `Data_easy/P000/lcam_front`;
- source class: `synthetic_ground_truth`;
- camera: 640x640 pinhole RGB-D at 10 Hz;
- reference channels: lossless metric float-depth and camera pose;
- license: CC BY 4.0;
- selected source frames: 48;
- verified public output: 212,565 surfels and 224,867 sparse-SDF voxels.

The artifact commits to both official archives before decoding:

| Role | Bytes | SHA-256 |
|---|---:|---|
| RGB | 517,844,269 | `9bea5fca9d0cf50105c7d34583d4d5db06e3715ef708262b4dfad763d34b17da` |
| Depth | 186,046,186 | `83e6e680297af35aa83d594ea3ed254bf71e9d9da7b26fee6d0ccb29f25ac104` |

The importer reads directly from those digest-checked ZIP archives. It does not
trust an extracted directory. Both download URLs are pinned to upstream revision
`0d2d145e973832742a2aaa04b7d2ebffc8d82817`. PNG dimensions, modality
alignment, pose count, frame ordering, decompressed entry sizes, and resource
bounds are checked before reconstruction. Float32 depth is converted to integer
millimeters from IEEE-754 bits without platform-dependent floating arithmetic.
Scientific pose decimals are converted directly into integer micrometers and
Q30 quaternions.

```bash
./scripts/reproduce-validation-origin.sh
cargo run --locked -q -p tessaryn-cli -- verify-validation-locus \
  apps/viewer-web/public/world/archviz-tiny-house-locus.json
```

## Real Sensor Stress Layer

Real datasets remain local and are not redistributed by this repository. The
`inspect-dataset` adapters produce content-bound receipts after checking source
layout and synchronization and hashing every regular file under the supplied
root. Symlinks, missing modalities, unsorted indexes, sample-count mismatches,
and resource-limit violations are rejected.

| Adapter | Sensors | Validation scope | Usage boundary |
|---|---|---|---|
| EuRoC MAV | stereo + IMU + motion ground truth | camera synchronization, high-rate IMU, trajectory reference | local noncommercial evaluation |
| KITTI raw | stereo + LiDAR + OXTS | outdoor modality alignment and vehicle motion | local noncommercial evaluation |
| ScanNet v2 | RGB + depth + pose + intrinsics | physical indoor RGB-D reconstruction input | terms-gated local evaluation |

EuRoC and KITTI remain useful sensor-stress benchmarks, but they are not
described as modern visual showcases: their releases date to 2016 and 2012.
ScanNet is access- and terms-gated. Product demonstrations should use separately
authorized first-party capture when physical grounding is part of the claim.
The redistributable TartanAir Locus remains useful for exact-ground-truth visual
regression and reproducible comparison, while real-dataset receipts provide
independent sensor-noise evidence.

```bash
cargo run --locked -p tessaryn-cli -- inspect-dataset euroc /data/euroc receipt.json
cargo run --locked -p tessaryn-cli -- inspect-dataset kitti /data/kitti-raw receipt.json
cargo run --locked -p tessaryn-cli -- inspect-dataset scannet /data/scannet-scene receipt.json
```

The browser camera reconstruction also has an opt-in photographic regression
against Middlebury's calibrated `templeRing` sequence. Keep the downloaded
dataset local, then run:

```console
TESSARYN_REAL_CAMERA_FIXTURE_DIR=/data/templeRing \
  npm --prefix apps/viewer-web run test:e2e -- \
  e2e/world-cell-real-camera.spec.ts --project=chromium
```

Continuous capture is separately gated with a real handheld TUM RGB-D sequence
at its native 30 Hz cadence. Extract the RGB movie into ordered JPEG or PNG
frames, then run:

```console
TESSARYN_HANDHELD_FRAME_DIR=/data/rgbd_dataset_freiburg1_desk2/frames \
TESSARYN_HANDHELD_REQUIRE_ROLLING=true \
  npm --prefix apps/viewer-web run test:e2e -- \
  e2e/world-cell-handheld-diagnostic.spec.ts --project=chromium
```

The gate requires geometry to appear while capture is still active, exercises
more than twelve admitted views, and then verifies that Finish preserves the
accepted reconstruction.

The canonical portfolio is generated by `tessaryn dataset-catalog` and shipped
to the browser at `validation/portfolio.json`. The viewer exposes the selected
lab profile, source class, sensor declaration, ground-truth channels,
archive footprint, and every validation adapter without merging their claims.
